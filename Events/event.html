/* I see. Event propagation is the process of how events travel through the DOM tree when an event occurs on an element¹². There are two phases of event propagation: capturing and bubbling². - Capturing phase: The event goes down from the window to the target element, passing through its ancestors². - Bubbling phase: The event goes up from the target element to the window, passing through its ancestors². Event listeners are functions that are executed when a specific event occurs on an element³. You can add event listeners to any element using the `addEventListener` method³. This method takes three arguments: the event type, the callback function, and a boolean value that indicates whether the listener should be executed in the capturing or bubbling phase³. For example, if you want to add a click event listener to a button element in the bubbling phase, you can write: ```javascript const button = document.querySelector("#myButton"); // select the button element button.addEventListener("click", () => { // add a click event listener console.log("Button clicked!"); // log a message to the console }, false); // indicate the bubbling phase ``` If you want to add the same listener in the capturing phase, you can change the last argument to `true`: ```javascript button.addEventListener("click", () => { console.log("Button clicked!"); }, true); // indicate the capturing phase ``` You can also stop the event propagation by using the `stopPropagation` method on the event object⁴. This method prevents the event from reaching further elements in the capturing or bubbling phase⁴. For example, if you want to stop the click event from bubbling up to the parent elements, you can write: ```javascript button.addEventListener("click", (event) => { // access the event object event.stopPropagation(); // stop the event propagation console.log("Button clicked!"); }, false); ``` I hope this explanation helps you understand the concepts of event propagation and event listener in JavaScript.
